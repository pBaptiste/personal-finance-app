Title: Personal Finance App

Tech Stack
* React Router - client
* Node/Express - server
* Mongodb - Database

Pages
* Login Page
* Sign Up Page
* Home/Overview Page
* Transactions Page
* Budget Page
* Pots Page
* Recurring Bills Page

Functionality
* Login/Sign Up: Users can Sign Up for an account by Name & email & password. Users will then have the ability to login with their credentials (email & password).
(Future feature: login using Social links)

* Navigation: The app possesses a persistent nav bar to navigate between each page. In the desktop view it is a side bar on the left. On Tablet and Mobile it is a bottom nav 

* Home/Overview Page
Made up of 7 components
1. A component with the header "current balance" that shows the total current amount that the user has saved.
2. A component with the header "income" that shows the current monthly	income
3. A component with the header "expense" that shows the current total monthly expense
4. A component with the header "pots". It shows the total amount a user has saved across their various "pots". This component also shows a list of all pots by rendering their header like "Concert ticket" and the amount like "$100". Last but not least is a link "see details" that takes users to their Pots page 
5. A transactions component that Lists the name of the each transaction, the amount and the date
6. A budgets component with a pie chart color coded and a list of the expenses with their name and amount
7. A recurring bills component with a list of recurring bills showing their names and amount. The component also has a button that takes you to the Recurring bills page.

* Transactions Page
The Transaction page shows a list of transactions in a table. Rows are divided into "Recipient/Sender", "Category", "Transaction Date", and "Amount". The table is paginated. At the top is a search bar to search for transactions, a select element for sorting by latest or earliest and a select element to sort by category. Users can create a new transaction by clicking the "add new transaction" button at the top. Users can also delete or update existing transactions

* Budget Page
The budget page has a component with a pie chart for all of the current spending and a spending summary with a list of spendings and their amounts. The second half of the page is a list of components each showing detailed information on each budget item. Each of these subcomponents show the name of the budget item, a line visualizing how much someone has spent versus how much they've budgeted for that budget item category and a list of the three most recent spending/transactions. Users can click on a button on the upper right of the page to open a model that will allow them to create a new budget item.

* Pots Page
The pots page shows a collection of pots components. Each component shows the name of the pot item like "Savings", the total amount in the pot, a line showing how much has been saved versus the target amount, a button to add money and one to withdraw. Each bringing up a model to make those mutations. Users can create a new pots item using a "Add new Pots" button at the top right.

* Recurring Bills Page
The recurring bills page has three main components. One shows the total amount for all recurring bills. Another component shows a how many of the bills have been paid this month and the total, how many upcoming bills to pay and the total, and how many are due soon and how much is due. The third component is a table with all recurring bills. This table shows The name, due date "Monthly -2nd" and amount for each bill. There is a search bar to search for bills, and a sorting by latest or earliest. Users can create a new recurring bill by clicking the "add new bill" button at the top. Users can also delete or update existing bills
